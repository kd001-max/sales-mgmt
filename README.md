# 🏭 销售生产管理系统 — 飞书集成版

> 基于飞书多维表格(Bitable) + 飞书审批 + Node.js 构建的 ERP 管理系统
> **全面支持飞书 PC 端和手机端使用**

---

## 📋 系统概述

本系统包含 **四大子系统**，完整覆盖从销售订单→生产计划→物料管控→销售发货的全流程：

| 子系统 | 核心功能 | 业务流程图 |
|--------|----------|-----------|
| 📋 **销售订货** | 订单下达 → 工程BOM评审 → 计划交期评审 → 业务确认 → 变更管理 | 订单全生命周期管理 |
| ⚙️ **生产计划** | 工单生成(MO) → 排程 → 周计划 → 每日计划 → 进度跟进 | 计划→执行→结案 |
| 📦 **物料管控** | 套料发放 → 退料 → 补料 → 完工入库 → 在线物料管理 | 物料全程追溯 |
| 🚚 **销售发货** | 发货单制作 → 部门审核 → 财务审批 → OQC → 装柜发货 | 发货全流程管控 |

## 🚀 部署步骤（30分钟可完成）

### 第一步：在飞书开放平台创建应用

1. 登录 [飞书开放平台](https://open.feishu.cn/) → 创建企业自建应用
2. 获取以下凭证：
   - `App ID`（应用标识）
   - `App Secret`（应用密钥）
3. 开启应用权限：
   - ✅ **多维表格**：`bitable:app`、`bitable:table`
   - ✅ **审批**：`approval:instance`
   - ✅ **消息**：`im:message`
   - ✅ **通讯录**：`contact:user`、`contact:department`
4. 发布应用（版本管理 → 创建版本 → 申请发布）

### 第二步：创建多维表格

1. 在飞书中新建一个「多维表格」，命名为 **销售生产管理系统**
2. 复制浏览器地址栏中的 `app_token`（URL中的一段32位字符串）
3. **自动建表**（推荐）：
   ```bash
   npm install
   cp .env.example .env
   # 编辑 .env，填入 FEISHU_APP_ID、FEISHU_APP_SECRET、BITABLE_APP_TOKEN
   npm run init-bitable
   ```
4. **手动建表**：参考 `bitable/schema.js` 中的字段定义，在飞书多维表格中逐表创建

> 需要创建的表清单（共16张）：
> 销售订单、产品BOM、物料主档、生产工单、生产排程、每日生产计划、周生产排程表、
> 生产发料单、退补料申请单、销售发货单、预测需求计划、订单变更通知单、联络单、
> 产品生产周期表、KPI统计、生产订单损耗统计表

### 第三步：部署后端服务

#### 方式一：部署到云服务器（推荐）

```bash
# 1. 上传项目到服务器
# 2. 安装依赖
npm install --production

# 3. 配置环境变量
# 编辑 .env 文件，参考 .env.example

# 4. 使用 PM2 启动（守护进程）
npm install -g pm2
pm2 start server.js --name sales-mgmt
pm2 save
pm2 startup

# 5. 配置 Nginx 反向代理（可选但有域名更好）
```

#### 方式二：部署到 Railway（免费）

1. GitHub 推送项目 → [Railway](https://railway.app/) 导入仓库
2. 设置环境变量（.env 中的全部变量）
3. 部署成功后获得 `https://xxx.railway.app` 域名

### 第四步：配置飞书应用

#### 配置自定义网页标签（使系统嵌入飞书使用）

在飞书开放平台 → 应用功能 → **网页**：

| 页面路径 | 页面名称 | PC端链接 | 移动端链接 |
|---------|---------|---------|-----------|
| `/` | 系统首页 | `https://你的域名/` | `https://你的域名/` |
| `/sales-order` | 销售订货 | `https://你的域名/sales-order` | 同上 |
| `/production-plan` | 生产计划 | `https://你的域名/production-plan` | 同上 |
| `/material-control` | 物料管控 | `https://你的域名/material-control` | 同上 |
| `/sales-shipment` | 销售发货 | `https://你的域名/sales-shipment` | 同上 |

配置完成后，在飞书PC端/手机端的「工作台」中即可看到该应用入口。

#### 配置机器人消息（通知提醒）

在 应用功能 → **机器人** 中开启，用于发送：
- 订单评审提醒 → 对应部门
- 生产计划变更通知
- 物料欠料预警
- 发货通知
- 每日生产计划

#### 配置审批流（订单评审、变更、退补料）

参考 `workflows/feishu-approval.json`，在飞书开放平台 **审批** 模块中创建4个审批定义：
1. `sales_order_review` - 销售订单评审
2. `order_change_notice` - 订单变更通知
3. `material_return_approval` - 退补料审批
4. `contact_note_approval` - 联络单审批

#### ⭐ 配置移动应用登录（用户身份识别）

> **推荐开启**，开启后系统能自动识别操作人的姓名、部门，实现操作日志可追溯

1. 在飞书开放平台 → **应用功能** → **移动应用登录** → 开启
2. 配置「回调地址」为 `https://你的域名/api/auth/callback`
3. 确保 `.env` 文件中配置了 `OAUTH_REDIRECT_URI`（与上方回调地址一致）

**开启后的效果**：
- 用户在电脑/手机上首次打开应用 → 自动跳转飞书授权页 → 一键授权登录
- 所有订单自动记录操作人（业务员=张三、工程评审人=李四）
- 页面右上角/顶部显示当前用户姓名和部门
- 后续操作（新建订单、评审、发料、发货）自动关联当前用户

## 🔄 核心业务流程

### 1️⃣ 销售订货流程

```
[客户下单] → [业务部录入订单] → [提交工程部评审]
    ↓
[工程部完成BOM (1-2天)] → [提交计划部评审]
    ↓
[计划部确认交期 (1个工作日)] → [业务部确认]
    ↓
[订单审核完成] → [进入生产环节]
```

### 2️⃣ 生产计划流程

```
[销售订单审核完成] → [生管生成生产工单(MO)]
    ↓
[生产排程] → [每周五发放次次周排程表]
    ↓
[每日生产计划（提前3天发放）] → [仓库备料]
    ↓
[生产执行] → [完工入库] → [结案]
```

### 3️⃣ 物料管控流程

```
[仓库套料发放] → [生产车间领料]
    ↓
[生产过程] → [退料（来料不良/制程损坏）]
    ↓
[补料申请（按比例多级审批）] → [完工入库]
```

### 4️⃣ 销售发货流程

```
[业务员制单] → [部门领导审核] → [财务审批]
    ↓
[仓库备货] → [OQC出货检验] → [装柜]
    ↓
[发货确认] → [开具放行条] → [完结]
```

## ⚙️ 环境变量说明

| 变量名 | 必须 | 说明 | 获取方式 |
|--------|------|------|---------|
| `FEISHU_APP_ID` | ✅ | 飞书应用AppID | 飞书开放平台 → 应用凭证 |
| `FEISHU_APP_SECRET` | ✅ | 飞书应用AppSecret | 飞书开放平台 → 应用凭证 |
| `BITABLE_APP_TOKEN` | ✅ | 多维表格的app_token | 多维表格URL中获取 |
| `OAUTH_REDIRECT_URI` | ⭐推荐 | 飞书OAuth登录回调地址 | 即 `https://你的域名/api/auth/callback` |
| `BOT_WEBHOOK_URL` | ❌ | 群机器人Webhook | 飞书群设置 → 群机器人 |
| `APPROVAL_CODE_*` | ❌ | 审批流程定义编码 | 飞书开放平台 → 审批定义 |
| `PORT` | ❌ | 服务端口（默认3000） | 自定义 |

## 🔧 系统维护

### 定时任务
系统内置以下定时任务（自动运行）：
| 时间 | 任务 | 说明 |
|------|------|------|
| 每天 09:00 | 每日生产计划提醒 | 推送当日计划到飞书群 |
| 每天 10:00 | 生管巡线提醒 | 督促到车间巡视 |
| 每周五 17:00 | 次次周计划发放提醒 | 提醒PC主管发放排程表 |
| 每月5日 09:00 | KPI统计提醒 | 提醒统计上月准时交付率 |
| 每月25日 10:00 | 销售预测会议提醒 | 提醒召开下月预测会议 |

### KPI指标
系统自动统计以下KPI：
- 📊 **订单准时交付率** = 准时交付订单数 / 总订单数
- 📊 **生产计划达成率** = 实际完成数 / 计划数
- 📊 **生产计划准确率** = 计划可执行率

## ❓ 常见问题

**Q: 飞书手机端打不开页面怎么办？**
A: 确保部署的服务器有公网域名（HTTPS），在飞书开放平台配置了正确的网页链接。

**Q: 多维表格数据无法读取？**
A: 检查 `.env` 中的 `BITABLE_APP_TOKEN` 是否正确，且飞书应用已发布并具有`bitable:app`权限。

**Q: 如何让不同部门看到不同的功能？**
A: 目前所有用户可访问全部页面。如需权限控制，可集成飞书用户身份认证和部门角色判断。

**Q: 可以直接在飞书手机上填写表单吗？**
A: 可以，所有表单已经过移动端适配，在飞书App内即可直接操作。
